  <action>
    Create scripts/wg-tunnel-up.sh as an executable bash script.

    Header:
    - #!/usr/bin/env bash
    - set -euo pipefail

    Variable declarations (using project pattern — required vars exit 64 on missing):
    - WG_INTERFACE default wg0
    - WG_PRIVATE_KEY required (no default) — exit code 64 if missing
    - WG_PEER_PUBLIC_KEY required — exit code 64 if missing
    - WG_ENDPOINT required (format HOST:PORT) — exit code 64 if missing
    - WG_LOCAL_IP default 10.8.0.2/32
    - WG_ALLOWED_IPS default 10.8.0.1/32
    - HANDSHAKE_TIMEOUT_SECS default 10
    - K8S_API_HOST default 10.8.0.1
    - K8S_API_PORT default 6443

    For required vars, use the project's exit-64 pattern: check the var is non-empty after the default expansion; if empty, echo to stderr and exit 64 (not exit 1 — per AGENTS.md "exits with code 64 for missing configuration").

    Implementation sections:

    1. Check wg is available (command -v wg); if not, install wireguard-tools via apt-get (sudo apt-get update + sudo apt-get install -y wireguard-tools); suppress output to avoid noise

    2. Create WireGuard interface:
       - sudo ip link add dev "$WG_INTERFACE" type wireguard
       - sudo ip address add "$WG_LOCAL_IP" dev "$WG_INTERFACE"

    3. Configure peer:
       - sudo wg set "$WG_INTERFACE" private-key <(printf '%s' "$WG_PRIVATE_KEY") peer "$WG_PEER_PUBLIC_KEY" endpoint "$WG_ENDPOINT" allowed-ips "$WG_ALLOWED_IPS"
       NOTE: Use <(printf '%s' "$WG_PRIVATE_KEY") — process substitution avoids writing the key to a temp file on disk; this is acceptable in bash

    4. Bring interface up:
       - sudo ip link set up dev "$WG_INTERFACE"

    5. Handshake polling loop (fail-closed):
       - Record start epoch with date +%s
       - Loop: check `sudo wg show "$WG_INTERFACE" latest-handshakes` — the output has format "peer_pubkey\ttimestamp_epoch"; handshake is complete when timestamp_epoch > 0 (grep for a tab followed by [1-9][0-9]* to detect a non-zero epoch)
       - On success: break
       - On timeout (elapsed > HANDSHAKE_TIMEOUT_SECS): echo FATAL message to stderr, dump wg show output for debug, exit 1
       - Poll every 0.25 seconds (sleep 0.25)
       - IMPORTANT: Never proceed if handshake not confirmed — this is the fail-closed gate

    6. API reachability check:
       - Use `timeout 5 bash -c "echo > /dev/tcp/$K8S_API_HOST/$K8S_API_PORT"` — exit 1 with FATAL message if it fails

    7. Final success echo: "WireGuard tunnel ready — API server reachable at $K8S_API_HOST:$K8S_API_PORT"

    Script must be executable: chmod +x is handled by git (set executable bit in the write).
    Add the executable permission via a comment in the action: the write tool should create the file with mode 755 (the executor should run chmod +x scripts/wg-tunnel-up.sh after writing).
  </action>

    Header:
    - #!/usr/bin/env bash
    - set -euo pipefail

    Variable declarations:
    - K8S_TOKEN required — exit 64 if missing (per AGENTS.md)
    - K8S_CA_CERT required — exit 64 if missing
    - K8S_API_SERVER default https://10.8.0.1:6443
    - K8S_NAMESPACE default solid-stats-staging
    - K8S_CLUSTER_NAME default k3s-staging
    - K8S_USER_NAME default ci-deployer
    - K8S_CONTEXT_NAME default ci-k3s-staging
    - KUBECONFIG default ${HOME}/.kube/config

    Implementation sections:

    1. Create kubeconfig directory: mkdir -p "$(dirname "$KUBECONFIG")"

    2. Write CA cert to temp file:
       - ca_file=$(mktemp)
       - trap 'rm -f "$ca_file"' EXIT
       - printf '%s' "$K8S_CA_CERT" > "$ca_file"

    3. Build kubeconfig using kubectl config set-* subcommands (NOT yaml templating):
       - kubectl config set-cluster "$K8S_CLUSTER_NAME" --certificate-authority="$ca_file" --server="$K8S_API_SERVER" --kubeconfig="$KUBECONFIG"
       - kubectl config set-credentials "$K8S_USER_NAME" --token="$K8S_TOKEN" --kubeconfig="$KUBECONFIG"
       - kubectl config set-context "$K8S_CONTEXT_NAME" --cluster="$K8S_CLUSTER_NAME" --user="$K8S_USER_NAME" --namespace="$K8S_NAMESPACE" --kubeconfig="$KUBECONFIG"
       - kubectl config use-context "$K8S_CONTEXT_NAME" --kubeconfig="$KUBECONFIG"
       NOTE: Never use --insecure-skip-tls-verify; the CA cert must be valid and 10.8.0.1 must be in the SANs (operator's responsibility per Plan 01)

    4. Verify authentication (fail-closed):
       - whoami_output=$(kubectl auth whoami --kubeconfig="$KUBECONFIG" 2>&1)
       - if echo "$whoami_output" | grep -q 'system:anonymous'; then echo FATAL to stderr; exit 1; fi
       - echo "Authenticated as: $whoami_output"

    5. Success echo: "kubeconfig ready at $KUBECONFIG"

    After writing, executor must run: chmod +x scripts/kubeconfig-setup.sh
  </action>
