<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host nginx (node) → obs pod | Untrusted-side public traffic terminates at host nginx and is proxied to grafana/glitchtip-web ClusterIPs; source IP seen by the pod is the node IP |
| obs namespace → app namespace (solid-stats-staging) | Prometheus/postgres-exporter cross into the app namespace to scrape; this is the only allowed obs→app path |
| compromised obs pod → lateral movement | A compromised pod in monitoring/error-tracking attempting to reach unrelated pods/namespaces |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Elevation/Spoofing | obs pod lateral reach | mitigate | default-deny ingress + egress on both obs namespaces; only named allow policies open specific flows |
| T-17-02 | Tampering | compromised obs pod → app DB | mitigate | egress to solid-stats-staging restricted to postgres-exporter:5432 and prometheus→rabbitmq:15692 only; no broad app-ns egress |
| T-17-03 | Information Disclosure | cross-ns log/metric exfil | mitigate | alloy egress limited to loki + k8s API; grafana egress limited to prometheus+loki |
| T-17-04 | Denial of Service | self-inflicted outage from wrong policy | accept | DNS allow ordered first; edge ipBlock + Alloy API egress confirmed empirically in 17-03 before relying on default-deny (validate-stack.sh green gate) |
| T-17-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this phase (RESEARCH §Package Legitimacy: none); built-in NetworkPolicy v1 API only |
