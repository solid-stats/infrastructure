    # Edge Bootstrap: nginx, TLS, and Firewall

    Intro paragraph (2-3 sentences): This is the one-time operator runbook for setting up the public staging edge — host nginx vhost, automatic TLS renewal, and host firewall — from the infrastructure repo. The bootstrap script is idempotent and safe to re-run. Live verification steps require SSH access to the VPS; offline structural checks run in CI via scripts/validate-edge.py.

    ## Prerequisites

    - SSH access to the staging VPS as root or with sudo.
    - Git clone of this repository on the VPS (or copy scripts/ and config/ to the host).
    - Port 80 inbound reachable from the public internet (required for ACME HTTP-01 challenge).
    - Port 443 inbound reachable (required for HTTPS access after bootstrap).
    - WireGuard wg0 interface up on the VPS (from Phase 6 operator-bootstrap.md).
    - ADMIN_EMAIL — a valid email for Let's Encrypt certificate registration.

    ## Offline Checks (CI / repo — no host required)

    Note: The following checks run in CI without touching the VPS. They validate the structure of the repo artifacts but NOT live nginx or certbot behavior.

    ```
    python3 scripts/validate-edge.py
    ```

    Expected output: five "ok:" lines covering nginx vhost, shell scripts, systemd units, bootstrap idempotency markers, and teardown script.

    ## Step 1: Clone / update the repo on the VPS

    Show: git clone or git pull command. The repo must be current so bootstrap-edge.sh installs the latest config files.

    ## Step 2: Run the bootstrap script

    ```bash
    ADMIN_EMAIL=your@email.com scripts/bootstrap-edge.sh
    ```

    - The script installs packages (certbot, nginx, ufw, curl, openssl), applies the nginx vhost, issues the initial TLS certificate via certbot webroot, installs systemd units, and configures ufw.
    - To skip certbot issuance (e.g. testing idempotency or no domain DNS yet):
      ADMIN_EMAIL=your@email.com SKIP_CERTBOT=1 scripts/bootstrap-edge.sh
    - The script is safe to re-run. Second run produces the same state as the first.

    ## Step 3: Operator-Only Live Verification

    Label each check clearly as OPERATOR-ONLY.

    3a. nginx config syntax:
    ```bash
    nginx -t
    ```
    Expected: "syntax is ok" and "test is successful". OPERATOR-ONLY — requires nginx installed.

    3b. Certbot renewal dry-run:
    ```bash
    certbot renew --dry-run
    ```
    Expected: "Congratulations, all simulated renewals succeeded" or "No renewals were attempted" (if cert is not yet due). OPERATOR-ONLY — requires certbot + issued cert + ACME connectivity.

    3c. systemd timer active:
    ```bash
    systemctl status certbot-renew.timer
    systemctl list-timers certbot-renew.timer
    ```
    Expected: Active (waiting), next trigger visible. OPERATOR-ONLY — requires systemd.

    3d. Renewal failure alerting:
    The OnFailure= wiring is structural (validate-edge.py checks it offline). To verify live:
    ```bash
    systemctl show -p OnFailure certbot-renew.service
    ```
    Expected output: OnFailure=certbot-renew-failure.target. OPERATOR-ONLY.

    3e. Firewall split-tunnel rules:
    ```bash
    ufw status verbose
    ```
    Expected: 22/tcp ALLOW Anywhere, 80/tcp ALLOW Anywhere, 443/tcp ALLOW Anywhere, 6443/tcp on wg0 ALLOW Anywhere. The 6443 rule must NOT appear without "on wg0". OPERATOR-ONLY — requires ufw.

    3f. Public HTTPS smoke check:
    ```bash
    curl -I https://stats-staging.solid-stats.ru/
    ```
    Expected: HTTP/2 200 (or appropriate app response). OPERATOR-ONLY — requires issued cert + public DNS.

    ## Step 4: Certificate Renewal Verification

    Note: Automatic renewal runs via certbot-renew.timer twice daily (01:00 and 13:00 UTC, randomized up to 1 hour). To verify the renewal pipeline manually:

    ```bash
    certbot renew --dry-run
    ```

    Check renewal failure alert path (verify logger fires on failure):
    ```bash
    journalctl -t certbot-alert --since today
    ```
    No output means no renewal failures today. Any entry indicates a failure — inspect with:
    ```bash
    journalctl -u certbot-renew.service -n 20
    ```

    ## Phase 11 Cutover Lever

    The nginx upstream is the Phase 11 traffic cutover lever. The exact line is in:

    ```
    config/nginx/sites-available/stats-staging.solid-stats.ru.conf
    ```

    Look for the line marked:
    ```nginx
    # CUTOVER: change this upstream to switch production traffic (Phase 11 lever)
    proxy_pass http://127.0.0.1:3000;
    ```

    To switch traffic: edit that single line to point to the legacy or new runtime upstream, then re-run bootstrap-edge.sh (or reload nginx manually with nginx -t first). Phase 11 documents this step in detail.

    ## Reversibility: Teardown

    Before Phase 11, the operator must prove the edge is reversible in isolation. Run teardown-edge.sh on the VPS to undo all bootstrap steps:

    ```bash
    scripts/teardown-edge.sh
    ```

    This removes:
    - nginx vhost symlink and config from /etc/nginx/sites-available/ and /etc/nginx/sites-enabled/
    - certbot-renew.timer, .service, and failure units from /etc/systemd/system/
    - certbot deploy hook from /etc/letsencrypt/renewal-hooks/deploy/
    - ufw rules for 80/tcp, 443/tcp, and 6443/tcp on wg0

    Preserved (NOT removed):
    - /etc/letsencrypt/ certificates (run certbot delete -d stats-staging.solid-stats.ru explicitly)
    - ufw allow 22/tcp SSH rule (removing it would lock out the operator)
    - ufw firewall itself (remains enabled with remaining rules)

    After teardown, verify:
    ```bash
    ufw status verbose             # 80/443/6443 rules must be absent
    systemctl list-timers          # certbot-renew.timer must be absent
    nginx -t                       # nginx config valid without the vhost
    ```

    ## Troubleshooting

    - **"FATAL: ADMIN_EMAIL is required"**: set ADMIN_EMAIL before running bootstrap-edge.sh.
    - **certbot "failed to connect to": port 80 not reachable** — verify Timeweb perimeter firewall allows port 80 inbound.
    - **"nginx -t: [emerg] unknown directive"**: config file has a syntax error — check config/nginx/sites-available/stats-staging.solid-stats.ru.conf for typos; run bootstrap again after fixing.
    - **"certbot-renew.timer failed"**: check journalctl -u certbot-renew.service -n 20 for the error; run certbot renew --dry-run manually to reproduce.
    - **ufw rule 6443 appears without "on wg0"**: this would expose the k3s API publicly — remove the incorrect rule with ufw delete allow 6443/tcp and re-run bootstrap-edge.sh.
    - **WireGuard wg0 interface not found** (ufw allow in on wg0 silently does nothing): ensure Phase 6 operator-bootstrap is complete and wg0 is up before running bootstrap-edge.sh.

    See also: docs/operator-bootstrap.md (Phase 6 WireGuard + RBAC bootstrap).
