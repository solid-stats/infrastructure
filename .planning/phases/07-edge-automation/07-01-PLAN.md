  <name>Task 2: nginx vhost config — config/nginx/sites-available/stats-staging.solid-stats.ru.conf (EDGE-01)</name>

  <action>
    Create the directory config/nginx/sites-available/ and write the nginx vhost config file.

    The file must have two server blocks:

    Block 1 — HTTP (port 80):
    - listen 80 and listen [::]:80 (IPv4 + IPv6).
    - server_name stats-staging.solid-stats.ru.
    - location /.well-known/acme-challenge/ { root /var/www/html; } — serves certbot webroot challenge files. Do NOT proxy this location.
    - location / { return 301 https://$host$request_uri; } — redirect all other HTTP to HTTPS.
    - A comment at the top of this block: "# HTTP: ACME challenge + redirect to HTTPS".

    Block 2 — HTTPS (port 443):
    - listen 443 ssl http2 and listen [::]:443 ssl http2.
    - server_name stats-staging.solid-stats.ru.
    - ssl_certificate /etc/letsencrypt/live/stats-staging.solid-stats.ru/fullchain.pem;
    - ssl_certificate_key /etc/letsencrypt/live/stats-staging.solid-stats.ru/privkey.pem;
    - TLS hardening:
      ssl_protocols TLSv1.2 TLSv1.3;
      ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
      ssl_prefer_server_ciphers on;
      ssl_session_timeout 1d;
      ssl_session_cache shared:SSL:50m;
    - HSTS header (required by validate-edge.py, ASVS V9):
      add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    - Upstream proxy block: location / { ... }. The proxy_pass line MUST be:
        # CUTOVER: change this upstream to switch production traffic (Phase 11 lever)
        proxy_pass http://127.0.0.1:3000;
      The comment "# CUTOVER:" on the line immediately before proxy_pass makes it the Phase 11 single-lever target. This is the line the operator edits to switch from staging k3s to legacy or new runtime.
    - Full proxy headers:
      proxy_set_header Host $host;
      proxy_set_header X-Real-IP $remote_addr;
      proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
      proxy_set_header X-Forwarded-Proto $scheme;
      proxy_http_version 1.1;
      proxy_set_header Connection "";
    - A comment at the top of this block: "# HTTPS: TLS termination + proxy to k3s server-2 upstream".
    - File header comment block:
      # stats-staging.solid-stats.ru — nginx vhost
      # Managed by: infrastructure repo (config/nginx/sites-available/)
      # Applied by: scripts/bootstrap-edge.sh (operator-run, not CI)
      # Phase 11 cutover lever: the proxy_pass line marked # CUTOVER:
      # Do not edit this file on the host directly — changes will be overwritten on re-bootstrap.

| T-7-01-02 | Spoofing | ACME challenge path — application receives challenge | mitigate | /.well-known/acme-challenge/ is served by nginx static root (/var/www/html), NOT proxied to server-2; the separate location block takes precedence over location / |
