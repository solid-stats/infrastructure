<objective>
Create the four systemd unit files (certbot-renew.service, .timer, -failure.target, -failure.service) and the certbot deploy-hook script (certbot-deploy-hook.sh). These deliver automatic TLS renewal (EDGE-02) and renewal-failure surfacing via journald (EDGE-03). All files are repo-stored; bootstrap-edge.sh (Plan 03) installs them to the VPS host.

Purpose: The systemd timer fires twice daily with a randomized delay (spread Let's Encrypt load). If certbot renewal fails, the OnFailure= chain triggers the failure service which calls `logger` to write a crit-priority entry to journald — operator-visible without any external monitoring infrastructure. The deploy-hook gates nginx reload on nginx -t to prevent a broken config from causing downtime.

Output: config/systemd/ directory with five files (per EDGE-02, EDGE-03).
</objective>

  <name>Task 1: systemd renewal units — certbot-renew.{service,timer} and failure chain (EDGE-02, EDGE-03)</name>
  <files>config/systemd/certbot-renew.service, config/systemd/certbot-renew.timer, config/systemd/certbot-renew-failure.target, config/systemd/certbot-renew-failure.service</files>
  <action>
    Create the config/systemd/ directory and write four unit files. These files are installed to /etc/systemd/system/ by bootstrap-edge.sh (Plan 03).

    FILE 1 — config/systemd/certbot-renew.service:
    Header comment: "# Source: config/systemd/certbot-renew.service — managed by infrastructure repo"
    [Unit]
    Description=Let's Encrypt Certificate Renewal
    After=network-online.target
    Wants=network-online.target
    OnFailure=certbot-renew-failure.target

    [Service]
    Type=oneshot
    ExecStart=/usr/bin/certbot renew --quiet
    User=root
    StandardOutput=journal
    StandardError=journal
    SyslogIdentifier=certbot-renew

    [Install]
    WantedBy=multi-user.target

    NOTE: The deploy hook (certbot-deploy-hook.sh) is installed separately to
    /etc/letsencrypt/renewal-hooks/deploy/ by bootstrap-edge.sh. certbot renew --quiet
    automatically invokes all scripts in that directory on successful renewal.
    Do NOT use --post-hook here — the deploy/ hook directory is the correct approach
    (it runs only on successful renewal, not on every renewal attempt).

    FILE 2 — config/systemd/certbot-renew.timer:
    Header comment: "# Source: config/systemd/certbot-renew.timer — managed by infrastructure repo"
    [Unit]
    Description=Let's Encrypt Certificate Renewal Timer
    Documentation=https://eff-certbot.readthedocs.io/en/stable/using.html#automated-renewals

    [Timer]
    OnCalendar=01:00
    OnCalendar=13:00
    RandomizedDelaySec=3600
    Unit=certbot-renew.service
    Persistent=true

    [Install]
    WantedBy=timers.target

    NOTE: Two OnCalendar= directives are valid in systemd — both fire. RandomizedDelaySec=3600
    spreads load up to 1 hour to avoid hammering Let's Encrypt at exactly 01:00/13:00 UTC.
    Persistent=true ensures the timer fires on next boot if the system was offline at the
    scheduled time.

    FILE 3 — config/systemd/certbot-renew-failure.target:
    Header comment: "# Source: config/systemd/certbot-renew-failure.target — triggered by OnFailure= in certbot-renew.service"
    [Unit]
    Description=Alert: Certbot Renewal Failed
    PartOf=certbot-renew.service

    [Install]
    WantedBy=multi-user.target

    FILE 4 — config/systemd/certbot-renew-failure.service:
    Header comment: "# Source: config/systemd/certbot-renew-failure.service — logs renewal failure to journald"
    [Unit]
    Description=Log Certbot Renewal Failure to journald
    PartOf=certbot-renew-failure.target

    [Service]
    Type=oneshot
    ExecStart=/bin/bash -c 'logger -t certbot-alert -p user.crit "Certbot renewal FAILED. Inspect: journalctl -u certbot-renew.service -n 20"'

    [Install]
    WantedBy=certbot-renew-failure.target

    NOTE: `logger -t certbot-alert -p user.crit` writes a crit-priority syslog entry with the
    tag "certbot-alert". This is visible via: journalctl -t certbot-alert --since today
    The message directs the operator to the renewal service logs for the actual failure reason.
    No external monitoring required — operator reads journald.
  </action>

<task type="auto">
  <name>Task 2: certbot deploy-hook script — config/systemd/certbot-deploy-hook.sh (EDGE-02)</name>
  <files>config/systemd/certbot-deploy-hook.sh</files>
  <action>
    Create config/systemd/certbot-deploy-hook.sh. This script is installed by bootstrap-edge.sh to /etc/letsencrypt/renewal-hooks/deploy/reload-nginx.sh. certbot calls all scripts in the deploy/ directory after each successful renewal.

    Header:
    #!/usr/bin/env bash
    set -euo pipefail

    Header comment block:
    # certbot-deploy-hook.sh — post-renewal nginx reload gate
    # Source: config/systemd/certbot-deploy-hook.sh — managed by infrastructure repo
    # Installed by bootstrap-edge.sh to: /etc/letsencrypt/renewal-hooks/deploy/reload-nginx.sh
    # certbot sets $RENEWED_DOMAINS before calling this hook.

    Implementation:

    TIMESTAMP=$(date +'%Y-%m-%d %H:%M:%S')
    RENEWED="${RENEWED_DOMAINS:-unknown}"
    echo "[$TIMESTAMP] [certbot-deploy-hook] Renewal hook triggered for: $RENEWED"

    Section 1 — nginx config validation (fail-closed gate):
    echo "[$TIMESTAMP] [certbot-deploy-hook] Validating nginx configuration..."
    if ! nginx -t 2>&1; then
      echo "[$TIMESTAMP] [certbot-deploy-hook] FATAL: nginx -t failed — refusing reload" >&2
      echo "[$TIMESTAMP] [certbot-deploy-hook] New certificate installed but nginx NOT reloaded" >&2
      echo "[$TIMESTAMP] [certbot-deploy-hook] Fix nginx config and run: systemctl reload nginx" >&2
      exit 1
    fi

    Section 2 — nginx reload:
    echo "[$TIMESTAMP] [certbot-deploy-hook] Reloading nginx..."
    systemctl reload nginx

    Section 3 — success log:
    TIMESTAMP=$(date +'%Y-%m-%d %H:%M:%S')
    echo "[$TIMESTAMP] [certbot-deploy-hook] Success — nginx reloaded with renewed certificate(s)"

    The script must be executable (bootstrap-edge.sh will chmod +x it after installing to the host). The repo file itself does not need to be executable — the bootstrap script handles permissions on the host copy.

    IMPORTANT: nginx -t must appear BEFORE systemctl reload nginx — this ordering is the safety gate (per EDGE-02). If nginx -t exits non-zero, the script exits 1 and certbot marks the renewal hook as failed (visible in journalctl -u certbot-renew.service). This triggers the OnFailure= chain from Plan 02's certbot-renew.service if the hook failure propagates.
  </action>
