Together they satisfy EDGE-04 (firewall: 80/443 public, 6443 WireGuard-only) and EDGE-05 (idempotent, re-runnable bootstrap that is proven reversible in isolation before it becomes the Phase 11 cutover lever).

The teardown makes the edge provably reversible: the operator can undo the full bootstrap and restore the pre-Phase-7 host state in one script invocation. This reversibility proof is required before Phase 11.

  <name>Task 1: Idempotent edge bootstrap — scripts/bootstrap-edge.sh (EDGE-04, EDGE-05)</name>
  <files>scripts/bootstrap-edge.sh</files>
  <action>
    Create scripts/bootstrap-edge.sh as an executable bash script following Phase 6 conventions (mirror wg-tunnel-up.sh and kubeconfig-setup.sh).

    Header:
    #!/usr/bin/env bash
    set -euo pipefail

    Header comment block:
    # bootstrap-edge.sh — idempotent host edge bootstrap (nginx vhost, certbot, systemd timer, ufw)
    # Run once on the VPS as root (or with sudo). Safe to re-run — all operations are idempotent.
    # See docs/edge-bootstrap.md for the full operator runbook.
    # Usage:
    #   ADMIN_EMAIL=ops@example.com scripts/bootstrap-edge.sh
    #   ADMIN_EMAIL=ops@example.com SKIP_CERTBOT=1 scripts/bootstrap-edge.sh  # skip cert issuance

    Determine REPO_ROOT: REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"

    Variable declarations — optional vars with safe defaults first:
    : "${DOMAIN:=stats-staging.solid-stats.ru}"
    : "${WEBROOT_PATH:=/var/www/html}"
    : "${NGINX_SITES_DIR:=/etc/nginx/sites-available}"
    : "${NGINX_SITES_ENABLED:=/etc/nginx/sites-enabled}"
    : "${SKIP_CERTBOT:=0}"

    Required vars — use the project exit-64 pattern (NOT ": ${VAR:?...}" — use explicit if/echo/exit 64 as in wg-tunnel-up.sh):
    if [[ -z "${ADMIN_EMAIL:-}" ]]; then
      echo "FATAL: ADMIN_EMAIL is required (Let's Encrypt registration email)" >&2
      exit 64
    fi

    Section structure (each section headed by: echo "=== [N]. <description> ==="):

    SECTION 1 — Package installation (idempotent via apt-get -y):
    - apt-get update -qq
    - apt-get install -y certbot nginx ufw curl openssl
    - Print versions (certbot --version, nginx -v) for the operator log.

    SECTION 2 — Webroot directory:
    - mkdir -p "$WEBROOT_PATH/.well-known/acme-challenge"
    - chmod -R 755 "$WEBROOT_PATH"

    SECTION 3 — nginx vhost config:
    - cp "$REPO_ROOT/config/nginx/sites-available/$DOMAIN.conf" "$NGINX_SITES_DIR/$DOMAIN.conf"
    - ln -sf "$NGINX_SITES_DIR/$DOMAIN.conf" "$NGINX_SITES_ENABLED/$DOMAIN.conf"
    - nginx -t (print result; exit 1 if fails with "FATAL: nginx config invalid — fix before continuing")
    - systemctl reload nginx || systemctl start nginx

    SECTION 4 — Initial TLS certificate issuance (skipped if cert exists or SKIP_CERTBOT=1):
    if [[ "${SKIP_CERTBOT}" == "1" ]]; then
      echo "SKIP_CERTBOT=1 — skipping certbot issuance"
    elif [[ -d "/etc/letsencrypt/live/$DOMAIN" ]]; then
      echo "Certificate already exists for $DOMAIN — skipping issuance"
    else
      certbot certonly \
        --authenticator webroot \
        --installer none \
        --no-eff-email \
        --agree-tos \
        --email "$ADMIN_EMAIL" \
        --webroot-path "$WEBROOT_PATH" \
        -d "$DOMAIN"
    fi

    SECTION 5 — certbot deploy hook:
    - mkdir -p /etc/letsencrypt/renewal-hooks/deploy
    - cp "$REPO_ROOT/config/systemd/certbot-deploy-hook.sh" /etc/letsencrypt/renewal-hooks/deploy/reload-nginx.sh
    - chmod +x /etc/letsencrypt/renewal-hooks/deploy/reload-nginx.sh

    SECTION 6 — systemd units:
    - for each unit file in certbot-renew.service certbot-renew.timer certbot-renew-failure.target certbot-renew-failure.service:
      cp "$REPO_ROOT/config/systemd/$unit" /etc/systemd/system/$unit
    - systemctl daemon-reload
    - systemctl enable --now certbot-renew.timer
    - systemctl is-enabled certbot-renew.timer (print result for operator log)

    SECTION 7 — ufw firewall rules (idempotent: ufw allow is a no-op if rule already exists):
    - ufw default deny incoming || true
    - ufw default allow outgoing || true
    - ufw allow 22/tcp comment 'SSH operator access' || true
    - ufw allow 80/tcp comment 'HTTP public + ACME challenges' || true
    - ufw allow 443/tcp comment 'HTTPS public' || true
    - ufw allow in on wg0 to any port 6443/tcp comment 'k3s API via WireGuard only' || true
    - ufw --force enable || true
    - echo "Firewall rules applied:"
    - ufw status verbose

    NOTE on ufw idempotency: ufw allow does NOT add duplicate rules if the rule already
    exists — it is safe to re-run. ufw --force enable is safe when ufw is already enabled.
    The `|| true` prevents set -e from aborting on first-run ufw "Skipping adding existing rule".

    SECTION 8 — Dry-run verification reminder (operator-only; not automated in CI):
    Print a summary box:
    echo "=== Bootstrap complete ==="
    echo "OPERATOR VERIFICATION REQUIRED (run these on the host):"
    echo "  1. nginx -t                                (validate config)"
    echo "  2. certbot renew --dry-run                 (verify renewal pipeline)"
    echo "  3. systemctl status certbot-renew.timer    (check timer active)"
    echo "  4. ufw status verbose                      (check split-tunnel rules)"
    echo "  5. curl -I https://$DOMAIN/                (smoke check public HTTPS)"
    echo "  See: docs/edge-bootstrap.md for full runbook"

    CRITICAL idempotency constraints:
    - mkdir -p everywhere (never bare mkdir)
    - ln -sf everywhere (force-overwrite, never bare ln)
    - All ufw allow calls have || true
    - ufw --force enable has || true
    - Section 4 checks for existing cert before attempting issuance
    - cp overwrites files (idempotent for configuration files)

    After creation, chmod +x scripts/bootstrap-edge.sh.
  </action>

<task type="auto">
  <name>Task 2: Edge teardown script — scripts/teardown-edge.sh (EDGE-05 reversibility)</name>
  <files>scripts/teardown-edge.sh</files>
  <action>
    Create scripts/teardown-edge.sh as an executable bash script. This script reverses every bootstrap step in the reverse order of bootstrap-edge.sh. It enables the operator to prove the edge is reversible in isolation before it becomes the Phase 11 cutover lever.

    Header:
    #!/usr/bin/env bash
    set -euo pipefail

    Header comment block:
    # teardown-edge.sh — reverses bootstrap-edge.sh, restores host to pre-Phase-7 state
    # Run on the VPS as root (or with sudo) to undo edge bootstrap.
    # IMPORTANT: This removes the nginx vhost, certbot timer, and ufw edge rules.
    # It does NOT remove TLS certificates from /etc/letsencrypt/ (certs are preserved
    # for potential re-bootstrap; run: certbot delete -d <DOMAIN> to remove certs explicitly).
    # Usage:
    #   scripts/teardown-edge.sh
    #   DOMAIN=stats-staging.solid-stats.ru scripts/teardown-edge.sh

    Variable defaults:
    : "${DOMAIN:=stats-staging.solid-stats.ru}"
    : "${NGINX_SITES_DIR:=/etc/nginx/sites-available}"
    : "${NGINX_SITES_ENABLED:=/etc/nginx/sites-enabled}"

    Section structure (each section headed by: echo "=== [N]. <description> ==="):

    SECTION 1 — Disable certbot timer and remove systemd units:
    - systemctl disable --now certbot-renew.timer 2>/dev/null || echo "certbot-renew.timer was not enabled — skipping"
    - systemctl stop certbot-renew.service 2>/dev/null || true
    - for each unit file: rm -f /etc/systemd/system/certbot-renew{.service,.timer,-failure.target,-failure.service}
    - systemctl daemon-reload
    - echo "certbot-renew timer and units removed"

    SECTION 2 — Remove certbot deploy hook:
    - rm -f /etc/letsencrypt/renewal-hooks/deploy/reload-nginx.sh
    - echo "certbot deploy hook removed"

    SECTION 3 — Remove nginx vhost:
    - rm -f "$NGINX_SITES_ENABLED/$DOMAIN.conf"   (remove symlink from sites-enabled)
    - rm -f "$NGINX_SITES_DIR/$DOMAIN.conf"        (remove config from sites-available)
    - if nginx -t 2>/dev/null; then systemctl reload nginx; else echo "warn: nginx -t failed after vhost removal — reload skipped"; fi
    - echo "nginx vhost for $DOMAIN removed"

    SECTION 4 — Remove ufw edge rules:
    - ufw delete allow 80/tcp 2>/dev/null || echo "ufw rule 80/tcp not found — skipping"
    - ufw delete allow 443/tcp 2>/dev/null || echo "ufw rule 443/tcp not found — skipping"
    - ufw delete allow in on wg0 to any port 6443/tcp 2>/dev/null || echo "ufw rule 6443/wg0 not found — skipping"
    NOTE: Do NOT remove ufw allow 22/tcp — removing SSH access would lock the operator out.
    NOTE: Do NOT run ufw disable — leave the firewall enabled with remaining rules intact.
    - echo "ufw edge rules removed (SSH rule preserved, firewall remains enabled)"
    - ufw status verbose

    SECTION 5 — Summary:
    echo "=== Teardown complete ==="
    echo "Removed: nginx vhost, certbot timer + units + deploy hook, ufw 80/443/6443-wg0 rules"
    echo "Preserved: /etc/letsencrypt/ certs (run 'certbot delete -d $DOMAIN' to remove)"
    echo "Preserved: ufw 22/tcp SSH rule (firewall remains enabled)"
    echo "OPERATOR VERIFICATION:"
    echo "  1. nginx -t                  (config valid without the vhost)"
    echo "  2. ufw status verbose        (confirm 80/443/6443 rules removed)"
    echo "  3. systemctl list-timers     (confirm certbot-renew.timer absent)"

    CRITICAL safety constraints:
    - Never remove ufw allow 22/tcp (operator lockout risk).
    - Never run ufw disable (leaves host open).
    - All rm -f calls have 2>/dev/null || echo "..." to handle missing files idempotently.
    - systemctl disable/stop have || true to handle not-enabled state.

    After creation, chmod +x scripts/teardown-edge.sh.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → host (root) | bootstrap-edge.sh and teardown-edge.sh run as root on the VPS; operator is the trust boundary; scripts are version-controlled and auditable before execution |
| public internet → ufw:80/443 | After bootstrap, ufw allows 80/443 inbound from any source; 6443 is blocked on public interfaces and allowed only on wg0 |
| WireGuard tunnel → ufw:6443 | 6443 is reachable only via interface wg0 (interface-specific rule); public IP cannot reach k3s API |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-7-03-01 | Elevation of Privilege | ufw 6443 exposed publicly due to missing interface qualifier | mitigate | Rule is "ufw allow in on wg0 to any port 6443/tcp" — interface qualifier wg0 is mandatory; validate-edge.py checks for "ufw allow in on wg0" in bootstrap-edge.sh; without wg0 qualifier, k3s API would be public |
| T-7-03-02 | Denial of Service | bootstrap-edge.sh locks out operator via ufw misconfiguration | mitigate | Script applies ufw allow 22/tcp BEFORE ufw --force enable; teardown never removes 22/tcp; firewall is never disabled during teardown; ssh access preserved through both operations |
| T-7-03-03 | Tampering | firewall rules applied in wrong order (deny before allow) | mitigate | ufw sets default deny incoming before adding allow rules; `ufw --force enable` fires last; all specific allow rules are present before enable; re-run is safe (idempotent) |
| T-7-03-04 | Availability | nginx -t failure prevents vhost from becoming active | mitigate | bootstrap-edge.sh runs nginx -t after cp and ln -sf and exits 1 if it fails; operator is prompted to fix the config error before proceeding; teardown conditionally skips reload if nginx -t fails post-removal |
| T-7-03-05 | Availability | SKIP_CERTBOT=0 and cert issuance fails (ACME unreachable) | mitigate | Section 4 checks for existing cert before attempting issuance; SKIP_CERTBOT=1 env var allows operator to skip issuance (e.g. first bootstrap in an air-gapped test); runbook documents pre-checking port 80 reachability |
| T-7-03-SC | Tampering | npm/pip/cargo installs | accept | Only apt-get installs (certbot, nginx, ufw, curl, openssl) — all approved in Package Legitimacy Audit in 07-RESEARCH.md; no npm/pip/cargo |
</threat_model>
