API support before the real retention policy is trusted. The Job is operator-gated (autonomous:false)
because it touches a live S3 bucket and records evidence that cannot be automated end-to-end.

Timeweb S3 accepts and parses lifecycle rules. The Job uses an ISOLATED prefix (s3-lifecycle-probe/)
and self-cleans — it never touches backups/postgres/ real objects.

  (subdirectory mirrors the restore-drill pattern; depth-1 glob exclusion prevents CD from
  accidentally scheduling the probe on every deploy)

and apply script from Plan 01 exist, so the operator cannot accidentally run the probe before
the policy artifacts are ready for review.

then waits for completion and records logs as evidence.

                4. API SUPPORT CHECK:
                   echo "=== API support: get-bucket-lifecycle-configuration ==="
                   get_output=$(aws --endpoint-url="$S3_ENDPOINT" s3api get-bucket-lifecycle-configuration \
                     --bucket "$S3_BUCKET" 2>&1 || true)
                   echo "$get_output"

                     echo "RESULT: lifecycle API NOT implemented on this endpoint" >&2

                     echo "RESULT: API implemented — no lifecycle config currently set (expected on first run)"
                   else
                     echo "RESULT: API implemented — existing lifecycle config found"
                   fi
                5. PUT-THEN-GET round-trip: put a 1-byte object under test prefix
                   echo "=== PUT test object: $TEST_KEY ==="
                   echo "probe" | aws --endpoint-url="$S3_ENDPOINT" s3 cp - "s3://$S3_BUCKET/$TEST_KEY" --only-show-errors
                6. HEAD-OBJECT: check x-amz-expiration header
                   echo "=== HEAD test object ==="
                   head_output=$(aws --endpoint-url="$S3_ENDPOINT" s3api head-object \
                     --bucket "$S3_BUCKET" --key "$TEST_KEY" 2>&1)
                   echo "$head_output"
                   if echo "$head_output" | grep -qi "expir"; then
                     echo "RESULT: x-amz-expiration PRESENT — lifecycle rule is recognized by endpoint"
                   else
                     echo "RESULT: x-amz-expiration ABSENT — endpoint may not compute expiry until a rule targets this prefix"
                     echo "NOTE: Apply a lifecycle rule to s3-lifecycle-probe/ prefix and re-run to observe expiry header"
                   fi
                7. CLEANUP: delete the test object
                   echo "=== CLEANUP: delete $TEST_KEY ==="
                   aws --endpoint-url="$S3_ENDPOINT" s3 rm "s3://$S3_BUCKET/$TEST_KEY" --only-show-errors
                   echo "probe cleanup complete"
                8. echo "S3-03 proof run complete — record logs as evidence"
              env:
                - name: S3_ENDPOINT, value: https://s3.twcstorage.ru

                - name: AWS_ACCESS_KEY_ID, valueFrom.secretKeyRef: name server-2-runtime, key S3_ACCESS_KEY_ID
                - name: AWS_SECRET_ACCESS_KEY, valueFrom.secretKeyRef: name server-2-runtime, key S3_SECRET_ACCESS_KEY
                - name: AWS_DEFAULT_REGION, value: ru-1
                - name: AWS_EC2_METADATA_DISABLED, value: "true"
              resources:

                limits: cpu 200m, memory 256Mi
              securityContext:
                allowPrivilegeEscalation: false
