| Alloy ServiceAccount → k8s API | read-only ClusterRole gates what Alloy can see |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-04 | Elevation of Privilege | Alloy ClusterRole | mitigate | read-only get/list/watch on pods/pods log/namespaces/events only; no write verbs, no secrets access (03-alloy-rbac.yaml, V4) |
| T-15-05 | Information Disclosure | log labels (request bodies / secrets) | mitigate | discovery.relabel + loki.process stage.label_keep allow exactly {namespace,pod,container,app,job}; everything else dropped before loki.write (LOG-02, V5) |
| T-15-06 | Denial of Service | Alloy self-log feedback loop | mitigate | discovery.relabel drop rule for __meta_kubernetes_namespace == "monitoring" (Pitfall 7) |
| T-15-07 | Tampering | cluster RBAC applied from CI | mitigate | rbac.create:false; ClusterRole only in operator-bootstrap k8s/staging/03-alloy-rbac.yaml; validate-obs-manifests.py forbids ClusterRole in k8s/observability/ (Pitfall 4) |
