  <action>
Create scripts/validate-stack.sh following RESEARCH §VAL-01 Script Design and the
existing harness conventions. Script style per AGENTS.md: shebang
#!/usr/bin/env bash, set -euo pipefail, header comment describing purpose/usage/flags/
required env. chmod +x it.

Behavior:
- Resolve SCRIPT_DIR via BASH_SOURCE so it finds sibling scripts regardless of cwd.
- Parse flags into a passthrough set: --quick (propagate to all three sub-scripts) and
  --public (propagate only to validate-phase-16.sh, which is the only one that accepts
  it — confirmed in its header). Reject unknown flags with exit 1, matching the
  sub-scripts' own flag handling.
- Preflight: `kubectl cluster-info --request-timeout=5s` must succeed or exit 1 with a
  FATAL message (RESEARCH §Pre-condition check) — the script needs a reachable cluster
  (WireGuard tunnel up).
- Run, in order, failing loudly on the first failure (set -e already aborts because the
  sub-scripts exit non-zero on failure; do NOT swallow their exit codes):
  1. Phase 13 metrics: invoke validate-phase-13.sh with K8S_NAMESPACE defaulting to
     monitoring (overridable via K8S_NAMESPACE_MONITORING), passing the --quick flag if set.
  2. Phase 15 logs: invoke validate-phase-15.sh with the same monitoring namespace,
     passing --quick if set.
  3. Phase 16 errors: invoke validate-phase-16.sh with K8S_NAMESPACE defaulting to
     error-tracking (overridable via K8S_NAMESPACE_ERROR), passing --quick and --public
     if set.
- Print a clear section banner before each sub-run and a single "FULL STACK VALIDATION
  PASSED" banner at the end (only reached if all three passed, since set -e aborts
  earlier on failure).
- Document required env in the header: GRAFANA_ADMIN_PASSWORD (metrics + logs Grafana
  datasource checks), GLITCHTIP_DSN (forced-error ingest in non-quick mode); optional
  SUPERUSER_TOKEN, GLITCHTIP_PUBLIC_URL, and the namespace overrides. Note that --quick
  skips the Grafana port-forward and the forced GlitchTip ingest (per the sub-scripts),
  which is the intended pre-policy pass; the full run is the post-policy pass.

Do NOT re-implement any assertion that lives in a sub-script — this is an orchestrator.
