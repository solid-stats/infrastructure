(3) Provision steps — fallocate -l 2G /swapfile; chmod 600; mkswap; swapon; append the `/swapfile swap swap defaults 0 0` line to /etc/fstab; set vm.swappiness=10 via /etc/sysctl.d/99-swap.conf + sysctl -p. (4) kubelet drop-in — create /var/lib/rancher/k3s/agent/etc/kubelet.conf.d/20-swap.conf with KubeletConfiguration failSwapOn:false and memorySwap.swapBehavior:NoSwap, then `systemctl restart k3s`. (5) Fallback — if `systemctl status k3s` shows a failSwapOn error after restart, set `kubelet-arg: fail-swap-on=false` in /etc/rancher/k3s/config.yaml instead (Pitfall 4). Note: the user's remote shell is fish — give the operator plain non-heredoc commands (write the drop-in with a redirect from a here-doc only if run as a single root bash invocation, or instruct creating the file via an editor; prefer `sudo install`/editor over fish-incompatible heredocs).
