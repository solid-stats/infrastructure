<objective>
Make the current staging workload manifests satisfy the Kubernetes safety
baseline or document explicit Phase 1 exceptions where the current k3s/image
contract prevents enforcement.
</objective>

<threat_model>
- T-01-04: Pods inherit the default ServiceAccount. Mitigation: add explicit
  ServiceAccounts or `automountServiceAccountToken: false` where API access is
  not needed.
- T-01-05: Unbounded stateful workloads starve the node. Mitigation: add
  resource requests and limits to PostgreSQL and RabbitMQ.
- T-01-06: NetworkPolicy assumptions are false on k3s. Mitigation: define
  policies only when enforceable, otherwise document `NetworkPolicy exception`.
</threat_model>

<action>
Add explicit ServiceAccounts for PostgreSQL, RabbitMQ, `server-2`,

the pod spec. Set each pod spec `serviceAccountName` to the matching
ServiceAccount name.
</action>
