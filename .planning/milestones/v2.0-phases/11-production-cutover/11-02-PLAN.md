Write docs/cutover.md following the edge-bootstrap.md runbook style (six-section structure:
overview → policy → pre-flight → procedure → rollback → post-cutover).

SECTION: Overview
Brief one-paragraph description: this is the Phase 11 production traffic cutover. The lever is
the nginx upstream server line in config/nginx/sites-available/stats-staging-solid-stats.conf
(the # CUTOVER: marker). Switching that one line (plus nginx -t + reload) moves all traffic
from legacy to the new k3s runtime. The mechanism is proven reversible by the Phase 7
bootstrap-edge/teardown-edge evidence.

SECTION: Policy
- The actual traffic flip is OPERATOR-EXECUTED, not automated. CI never runs cutover.sh.
- The green-diff gate is COVERAGE/INTEGRITY ONLY — NOT value-equality. The new parser
  (server-2/replay-parser-2) deliberately diverges from legacy in computed stat values
  (memory: legacy-vs-new-parser-non-identical). strict_failures = missing matches/players,
  parser errors, aggregate totals outside declared tolerance. Intended value differences are
  allowlisted and human-reviewed. docs/diff-readiness.md defines the full contract.
- CUT-05 (weighted/blue-green gradual shift) is DEFERRED to v2.x.
- Do NOT execute the cutover until both gates pass and the operator has reviewed the diff output.

SECTION: Pre-flight Gates (CUT-03)
Gate 1 — Fresh verified backup:
  docs/backup-gate.md must contain "Status: verified" with a backup younger than the planned
  ingest window. Run a fresh backup if the last one is stale:
    kubectl -n solid-stats-staging create job postgres-backup-preflight-$(date -u +%Y%m%d%H%M%S) \
      --from=cronjob/postgres-backup
  Confirm the Job completes, then update docs/backup-gate.md with the new backup ID.

Gate 2 — Green diff coverage (NOT equality):
  docs/diff-readiness.md must contain "strict_failures: 0" — meaning no missing players,
  no missing matches, no parser errors, no unexplained aggregate total deviations.
  Allowlisted known differences (value divergence from the rewrite) do NOT block the gate.
  The operator must have reviewed the diff output before running cutover.
  To record the gate evidence: add the line "strict_failures: 0" to docs/diff-readiness.md
  under a "## Cutover Gate Evidence" section after the human review is complete.

Gate 3 — Live edge proven reversible:
  Phase 7 bootstrap→teardown→re-bootstrap cycle was executed and passed (see docs/edge-bootstrap.md).
  The vhost .bak backup exists on the VPS (created by bootstrap-edge.sh). Confirm:
    ls -la /etc/nginx/sites-available/stats-staging-solid-stats.conf.bak

Gate 4 — Smoke check target defined:
  NEW_UPSTREAM is known and the target runtime is healthy in k3s:
    kubectl -n solid-stats-staging rollout status deployment/server-2
    kubectl -n solid-stats-staging get svc   # confirm the target ClusterIP:port

SECTION: Timing — How Long to Accumulate Before Flipping
Recommended approach before executing the cutover:
1. Run at least one full controlled ingest (docs/full-run.md) with the new runtime active.
2. Let the new runtime serve traffic in parallel for at least 24h (observe error rates in logs).
3. Confirm the diff gate passes (strict_failures: 0) after reviewing the full-run output.
4. Only then declare both gates met and proceed.
There is no hard minimum; the operator decides when confidence is sufficient. The cutover is
reversible within minutes if the smoke check or post-cutover monitoring reveals a problem.

SECTION: Cutover Procedure (CUT-01)
Pre-flight on the VPS:
  Confirm gate docs are met (sections above), then:
    NEW_UPSTREAM=<target_address:port> scripts/cutover.sh

  For a dry-run preview (does NOT touch nginx):
    DRY_RUN=1 NEW_UPSTREAM=<target_address:port> scripts/cutover.sh

What cutover.sh does:
  1. Checks "Status: verified" in docs/backup-gate.md.
  2. Checks "strict_failures: 0" in docs/diff-readiness.md.
  3. Backs up the live vhost to /etc/nginx/sites-available/stats-staging-solid-stats.conf.cutover.bak
  4. Replaces the server line at the # CUTOVER: marker with NEW_UPSTREAM.
  5. Runs nginx -t (fail-closed — exits without reloading on invalid config).
  6. Reloads nginx.
  7. Smoke-checks curl -fsS -I https://stats-staging.solid-stats.ru/ (3 retries).
  8. AUTO-ROLLBACK if smoke check fails (restores backup + reloads nginx).

Post-cutover immediate checks (OPERATOR-ONLY — live VPS):
  nginx -t
  curl -I https://stats-staging.solid-stats.ru/
  kubectl -n solid-stats-staging logs deployment/server-2 --since=5m | tail -20

SECTION: Rollback (CUT-02)
If you need to revert after a successful cutover:
  ROLLBACK METHOD 1 — use cutover.sh rollback function:
    # Edit cutover.sh: call rollback() directly, or:
    cp /etc/nginx/sites-available/stats-staging-solid-stats.conf.cutover.bak \
       /etc/nginx/sites-available/stats-staging-solid-stats.conf
    nginx -t && systemctl reload nginx

  ROLLBACK METHOD 2 — re-run bootstrap-edge.sh:
    # Edit config/nginx/sites-available/stats-staging-solid-stats.conf:
    # restore the original server line (10.43.94.103:3000), then:
    ADMIN_EMAIL=ops@example.com SKIP_UFW=1 scripts/bootstrap-edge.sh

  Verify rollback:
    curl -I https://stats-staging.solid-stats.ru/   # should return from legacy upstream
    nginx -t

  The Phase 7 teardown evidence (docs/edge-bootstrap.md Reversibility section) proves that
  vhost backup → restore → nginx reload is a working, tested recovery path.

def validate_cutover_artifacts() -> None:
    """Offline gate for CUT-01..04: assert cutover script and runbook exist with required markers."""
    # Script existence and bash syntax

    require(script_path.is_file(), "scripts/cutover.sh missing (CUT-01)")
    result = run(["bash", "-n", str(script_path)])
    require(result.returncode == 0,
            f"scripts/cutover.sh failed bash -n: {result.stderr.strip()}")

    content = script_path.read_text()
    require("set -euo pipefail" in content,
            "scripts/cutover.sh missing 'set -euo pipefail'")
    require("exit 64" in content,
            "scripts/cutover.sh missing 'exit 64' (required env var guard)")

            "scripts/cutover.sh missing backup gate grep for 'Status: verified' (CUT-03)")

            "scripts/cutover.sh missing diff coverage gate grep for 'strict_failures: 0' (CUT-03)")

    require("rollback" in content,
            "scripts/cutover.sh missing rollback function (CUT-02)")
    # nginx fail-closed gate
    require("nginx -t" in content,

            "docs/cutover.md missing reference to docs/backup-gate.md")
