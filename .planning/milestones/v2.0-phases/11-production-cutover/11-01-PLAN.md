<objective>
Deliver the 4-gate reversible production cutover script (scripts/cutover.sh).

The script is the operator-facing mechanism for switching nginx production traffic from legacy
to the new runtime. It gates on (1) a fresh verified backup and (2) a green coverage diff before
touching nginx, backs up the live vhost, switches the upstream server line at the # CUTOVER:
marker, validates config (fail-closed), reloads nginx, runs a smoke-check curl, and auto-rolls
back on smoke failure. A rollback() function restores the vhost backup independently.

Purpose: CUT-01 (single reversible switch), CUT-02 (tested rollback), CUT-03 (backup + diff
gates), CUT-04 (post-cutover smoke check). The actual live traffic flip is OPERATOR-gated
(autonomous:false in plan 11-02); this plan builds and offline-proves the mechanism.

Output: scripts/cutover.sh
</objective>

Create scripts/cutover.sh following the exact conventions of scripts/bootstrap-edge.sh and
scripts/teardown-edge.sh: #!/usr/bin/env bash, set -euo pipefail, required() helper with
exit 64, default-with-override vars via ${VAR:=default}.

Required env vars (required() + exit 64 if unset):
  NEW_UPSTREAM   — the target upstream address:port (e.g. 192.168.1.10:8080)

Optional vars with defaults (${VAR:=default}):
  VHOST_HOST     — defaults to "stats-staging.solid-stats.ru"
  VHOST_CONF     — defaults to "/etc/nginx/sites-available/stats-staging-solid-stats.conf"
  BACKUP_GATE_FILE    — defaults to "docs/backup-gate.md"
  DIFF_GATE_FILE      — defaults to "docs/diff-readiness.md"
  SMOKE_RETRIES  — defaults to 3
  SMOKE_DELAY    — defaults to 5 (seconds between retries)
  DRY_RUN        — defaults to "" (set to "1" to print plan without executing)

The script structure:

SECTION 1 — env + required() + var defaults (as above).

SECTION 2 — gate checks (both must pass before touching nginx):
  Gate A (CUT-03 backup): Mirror start-controlled-full-run.sh exactly:
    if ! grep -q "Status: verified" "${BACKUP_GATE_FILE}"; then
      echo "FATAL: backup gate not verified in ${BACKUP_GATE_FILE} — run a fresh backup first" >&2; exit 1; fi
  Gate B (CUT-03 diff/coverage): NOT an equality gate — checks for coverage evidence marker:
    if ! grep -q "strict_failures: 0" "${DIFF_GATE_FILE}"; then
      echo "FATAL: diff coverage gate not met in ${DIFF_GATE_FILE} (strict_failures: 0 not found)" >&2
      echo "  The green-diff gate is coverage/integrity only — see docs/diff-readiness.md" >&2; exit 1; fi

SECTION 3 — vhost backup (reversibility):
  BAK_VHOST="${VHOST_CONF}.cutover.bak"
  If the live vhost does not exist, exit 1 with an error.
  Always write the backup (cp -p "${VHOST_CONF}" "${BAK_VHOST}") even on re-run — overwrite
  any prior backup so it always reflects the state before THIS run's edit.

SECTION 4 — rollback() function (CUT-02):
  Define rollback() before any nginx-touching code so it can be called from the smoke-check
  failure path and manually by the operator. rollback() must:
    1. Verify BAK_VHOST exists; exit 1 if not (cannot roll back without backup).
    2. cp "${BAK_VHOST}" "${VHOST_CONF}" — restore the backup byte-for-byte.
    3. nginx -t (fail-closed); if nginx -t fails after restore, print FATAL and exit 1.
    4. systemctl reload nginx.
    5. Print "ROLLBACK COMPLETE — upstream restored from backup" and the backup path.
  The rollback function must NOT silently succeed; every step is checked.

SECTION 5 — upstream switch (CUT-01):
  The # CUTOVER: line in the vhost is the lever. The upstream server line immediately follows it.
  Use sed to replace the server line in the upstream block:
    sed -i "s|^\( *\)server [^;]*;|\1server ${NEW_UPSTREAM};|" "${VHOST_CONF}"
  After sed, verify the new upstream address appears in the file:
    if ! grep -q "${NEW_UPSTREAM}" "${VHOST_CONF}"; then
      echo "FATAL: sed did not write NEW_UPSTREAM to vhost — aborting" >&2; exit 1; fi

SECTION 6 — nginx -t gate (fail-closed):
  Run nginx -t. If it fails: print the error, call rollback(), exit 1. Never reload on a
  bad config. This mirrors the bootstrap-edge.sh fail-closed pattern exactly.

SECTION 7 — nginx reload:
  systemctl reload nginx. If reload fails despite nginx -t passing, call rollback(), exit 1.

SECTION 8 — smoke check with auto-rollback (CUT-04):
  Use curl -fsS -o /dev/null -w '%{http_code}' to capture the HTTP status code explicitly.
  curl -fsS exits non-zero on 4xx/5xx and on transport errors; the -w '%{http_code}' output
  is captured for logging only.

  smoke_ok=0
  for i in $(seq 1 ${SMOKE_RETRIES}); do
    http_code=$(curl -fsS -o /dev/null -w '%{http_code}' \
                  "https://${VHOST_HOST}/" --max-time 10 2>/dev/null) || true
    if [[ "$http_code" =~ ^[23] ]]; then   # 2xx or 3xx = success
      echo "Smoke check passed (HTTP ${http_code}) — new upstream ${NEW_UPSTREAM} responding"
      smoke_ok=1
      break
    fi
    echo "Smoke check attempt $i/${SMOKE_RETRIES} failed (HTTP ${http_code:-ERR}) — waiting ${SMOKE_DELAY}s..." >&2
    sleep "${SMOKE_DELAY}"
  done
  if [[ "$smoke_ok" -eq 0 ]]; then
    echo "FATAL: smoke check failed after ${SMOKE_RETRIES} attempts — AUTO-ROLLBACK" >&2
    rollback
    exit 1
  fi
  # If smoke_ok=1, fall through to SECTION 9 (success banner) and exit 0.

  Control flow summary:
    - Smoke passes on attempt i (1..SMOKE_RETRIES): smoke_ok=1, break, fall through to
      SECTION 9, script exits 0.
    - All SMOKE_RETRIES attempts fail: smoke_ok remains 0; after the loop, rollback() is
      called and the script exits 1. SECTION 9 is never reached.

SECTION 9 — success banner (reached only on smoke_ok=1):
  Print the previous upstream (extracted from the .bak file with grep for the server line),
  the new upstream, the backup path, and the rollback command to undo manually if needed.
  Never echo any value that could be a secret (only upstream addresses and file paths).
  Exit 0 at the end of this section.

DRY_RUN support (BLOCKER #1 fix — gates are NOT bypassed in DRY_RUN mode):
  DRY_RUN=1 STILL enforces Gate A and Gate B. If either gate is unmet, the script prints
  the FATAL error to stderr and exits 1, EVEN in DRY_RUN mode. This proves the gates work
  before a live operator run.
  Only implementation steps (SECTIONS 3–9: backup, upstream switch, nginx -t, reload,
  smoke-check curl, success banner) are replaced by "[DRY-RUN] would: <action>" prints.
  Gate checks (SECTION 2) always execute with full refusal logic regardless of DRY_RUN.

  Implement as: after SECTION 2 (gates pass), check DRY_RUN before each implementation step:
    if [[ "${DRY_RUN:-}" == "1" ]]; then
      echo "[DRY-RUN] would: cp -p ${VHOST_CONF} ${BAK_VHOST}"
      echo "[DRY-RUN] would: sed upstream switch to ${NEW_UPSTREAM}"
      echo "[DRY-RUN] would: nginx -t"
      echo "[DRY-RUN] would: systemctl reload nginx"
      echo "[DRY-RUN] would: curl smoke-check https://${VHOST_HOST}/"
      echo "[DRY-RUN] gates PASSED — script would proceed to live flip"
      exit 0
    fi
  This early-exit after gate checks and before the first mutation means DRY_RUN previews
  the FULL plan only if the operator has satisfied both pre-flight gates.

SELF_TEST support (WARNING #1 fix — rollback() exercised in isolation):
  SELF_TEST=1 tests the rollback() function offline WITHOUT touching the live nginx config.
  Add a self-test path after the rollback() function definition (SECTION 4):

    if [[ "${SELF_TEST:-}" == "1" ]]; then
      echo "=== SELF_TEST: exercising rollback() on a temp vhost copy ==="
      TMP_VHOST=$(mktemp /tmp/cutover-selftest-XXXXXX.conf)
      # Copy the real vhost to a temp file so rollback() has something to restore
      cp "${VHOST_CONF}" "${TMP_VHOST}"
      # Simulate the backup that rollback() expects
      cp "${TMP_VHOST}" "${TMP_VHOST}.cutover.bak"
      # Simulate a bad switch: corrupt the temp file
      echo "server CORRUPTED:9999;" >> "${TMP_VHOST}"
      # Temporarily override VHOST_CONF and BAK_VHOST to point at temp files
      ORIG_VHOST_CONF="${VHOST_CONF}"
      ORIG_BAK_VHOST="${BAK_VHOST}"
      VHOST_CONF="${TMP_VHOST}"
      BAK_VHOST="${TMP_VHOST}.cutover.bak"
      # Call rollback — it must restore TMP_VHOST from TMP_VHOST.cutover.bak
      rollback
      # Assert: temp file is byte-restored (matches the original pre-corruption copy)
      if ! diff -q "${TMP_VHOST}" "${TMP_VHOST}.cutover.bak" >/dev/null 2>&1; then
        echo "SELF_TEST FAILED: rollback did not restore file to backup state" >&2
        rm -f "${TMP_VHOST}" "${TMP_VHOST}.cutover.bak"
        VHOST_CONF="${ORIG_VHOST_CONF}"; BAK_VHOST="${ORIG_BAK_VHOST}"
        exit 1
      fi
      echo "SELF_TEST PASSED: rollback() correctly restored the temp vhost from backup"
      rm -f "${TMP_VHOST}" "${TMP_VHOST}.cutover.bak"
      VHOST_CONF="${ORIG_VHOST_CONF}"; BAK_VHOST="${ORIG_BAK_VHOST}"
      exit 0
    fi

  SELF_TEST=1 skips SECTION 2 gate checks (it tests rollback mechanics, not gate docs),
  skips all live nginx mutations, and exits 0 on success / 1 on failure. Safe to run on
  any host that has a readable VHOST_CONF (even a non-live copy).

The script must NOT be placed in any directory reachable by the CD depth-1 glob
(k8s/staging/*.yaml). It lives in scripts/ which is correct — validate-staging.py enforces
DRILL-04 only on k8s/staging/*.yaml; scripts/ is never auto-applied by CD.
