<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions → staging environment secrets | Deploy job requires `environment: staging`; secrets are only exposed to that environment; PR forks do not have access to environment secrets |
| CI runner → k3s API via WireGuard | All manifest operations flow through the authenticated kubeconfig over the WireGuard tunnel |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-6-04-01 | Information Disclosure | SA token / WireGuard private key in CI logs | mitigate | GitHub environment secrets are masked in logs; scripts never echo token or key values; K8S_TOKEN passed via --token= in kubectl config (not env var printed to stdout) |
| T-6-04-02 | Spoofing | PR fork accessing environment secrets | mitigate | `environment: staging` on deploy and dry-run jobs — GitHub does not expose environment secrets to workflows triggered by pull_request from forks; only org members with staging environment access |
| T-6-04-03 | Denial of Service | Concurrent deploys racing | mitigate | concurrency group: infrastructure-staging-deploy, cancel-in-progress: false — second deploy waits; no two deploys apply manifests simultaneously; prevents StatefulSet race conditions |
| T-6-04-04 | Tampering | Deploy proceeding over dead/incomplete tunnel | mitigate | wg-tunnel-up.sh is the first step before kubeconfig-setup.sh and before any kubectl; exits 1 on handshake timeout and TCP connect failure; deploy job fails if any preceding step fails (set -euo pipefail + workflow step error propagation) |
| T-6-04-05 | Elevation of Privilege | CI applying RBAC manifests (01-ci-rbac.yaml) | mitigate | find glob excludes 01-ci-rbac.yaml from both dry-run and deploy apply; CI SA token does not have permissions to manage Roles/RoleBindings anyway (not in Role rules); double defense-in-depth |
| T-6-04-06 | Tampering | deploy-staging.sh retained with SSH references | mitigate | Task 2 deletes the file; grep check confirms no code file references it after deletion |
| T-6-04-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan; wireguard-tools install is in wg-tunnel-up.sh (already approved in Package Legitimacy Audit); Python is pre-installed on ubuntu-latest |
</threat_model>
