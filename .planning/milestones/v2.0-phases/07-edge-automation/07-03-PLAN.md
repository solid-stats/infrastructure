<objective>
Create the idempotent adopt-reconcile bootstrap script (scripts/bootstrap-edge.sh) and its
reversibility counterpart (scripts/teardown-edge.sh).

The adopt-not-build reality (per D-8): the live edge already exists and serves traffic. Bootstrap
must:
  1. Back up the live vhost before overwriting (so teardown can restore the exact original).
  2. Replace the live vhost with the repo copy, gating on nginx -t before reload.
  3. Install the certbot.service.d drop-in + daemon-reload (enables OnFailure= surfacing).
  4. Install the deploy hook to /etc/letsencrypt/renewal-hooks/deploy/.
  5. Apply ufw rules (80/443 public, 6443 on wg0 only, per D-7).
  6. Skip cert issuance if lineage already exists (per D-6).

Teardown reverses all steps: restores the backed-up live vhost, removes the drop-in, removes
the deploy hook, removes ufw 80/443/6443-wg0 rules (preserves SSH and /etc/letsencrypt/).

Purpose: Completes the EDGE-04 (firewall) and EDGE-05 (idempotent + reversible) requirements.
Together with Plans 01 and 02, this makes the full edge adopt/reconcile cycle executable.

Output: scripts/bootstrap-edge.sh, scripts/teardown-edge.sh (per EDGE-04, EDGE-05).
</objective>

  <name>Task 1: Idempotent adopt-reconcile bootstrap — scripts/bootstrap-edge.sh (EDGE-04, EDGE-05, D-6..D-9)</name>

  <action>
    Create scripts/bootstrap-edge.sh as an executable bash script mirroring Phase 6
    conventions from wg-tunnel-up.sh and kubeconfig-setup.sh.

    Header:
    #!/usr/bin/env bash
    set -euo pipefail

    Header comment block:
    # bootstrap-edge.sh — idempotent adopt-reconcile for the staging host edge
    # Backs up the live vhost, installs the repo vhost + certbot drop-in + deploy hook + ufw rules.
    # Run on the VPS as root (or with sudo). Safe to re-run — all ops are idempotent.
    # Usage:
    #   ADMIN_EMAIL=ops@example.com scripts/bootstrap-edge.sh
    #   ADMIN_EMAIL=ops@example.com SKIP_UFW=1 scripts/bootstrap-edge.sh   # skip ufw changes
    # See docs/edge-bootstrap.md for the full operator runbook.

    REPO_ROOT: REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"

    Optional vars with defaults:
    : "${DOMAIN:=stats-staging.solid-stats.ru}"
    : "${WEBROOT_PATH:=/var/www/html}"
    : "${NGINX_SITES_DIR:=/etc/nginx/sites-available}"
    : "${NGINX_SITES_ENABLED:=/etc/nginx/sites-enabled}"
    : "${SKIP_UFW:=0}"

    Required vars — use explicit if/echo/exit 64 (project pattern, per AGENTS.md):
    if [[ -z "${ADMIN_EMAIL:-}" ]]; then
      echo "FATAL: ADMIN_EMAIL is required (Let's Encrypt registration email)" >&2
      exit 64
    fi

    Section structure (each section headed by: echo "=== [N]. <description> ==="):

    SECTION 1 — Package check (idempotent via apt-get -y):
    apt-get update -qq
    apt-get install -y certbot nginx ufw curl openssl
    Print versions for the operator log.

    SECTION 2 — Webroot directory:
    mkdir -p "$WEBROOT_PATH/.well-known/acme-challenge"
    chmod -R 755 "$WEBROOT_PATH"

    SECTION 3 — Adopt / reconcile nginx vhost (per D-8):
    VHOST_CONF="$NGINX_SITES_DIR/stats-staging-solid-stats.conf"
    REPO_VHOST="$REPO_ROOT/config/nginx/sites-available/stats-staging-solid-stats.conf"
    BAK_VHOST="${VHOST_CONF}.bak"

    # Back up the live file before overwriting (only if not already backed up)
    if [[ -f "$VHOST_CONF" && ! -f "$BAK_VHOST" ]]; then
      echo "Backing up live vhost to $BAK_VHOST..."
      cp "$VHOST_CONF" "$BAK_VHOST"
    elif [[ -f "$BAK_VHOST" ]]; then
      echo "Backup already exists at $BAK_VHOST — skipping backup step"
    fi

    # Install repo vhost (idempotent: cp overwrites)
    cp "$REPO_VHOST" "$VHOST_CONF"
    # Symlink into sites-enabled (idempotent: ln -sf)
    ln -sf "$VHOST_CONF" "$NGINX_SITES_ENABLED/stats-staging-solid-stats.conf"

    # nginx -t gate — fail closed; never reload on invalid config
    echo "Validating nginx configuration..."
    if ! nginx -t 2>&1; then
      echo "FATAL: nginx config invalid after vhost install — restoring backup" >&2
      if [[ -f "$BAK_VHOST" ]]; then
        cp "$BAK_VHOST" "$VHOST_CONF"
        # Re-validate AFTER restore before any reload — never reload an unvalidated config
        if ! nginx -t; then
          echo "FATAL: restored vhost still invalid — refusing reload to avoid breaking the live edge" >&2
          exit 1
        fi
      fi
      exit 1
    fi
    systemctl reload nginx || systemctl start nginx

    SECTION 4 — TLS certificate (skip if lineage exists, per D-6, D-8):
    if [[ -d "/etc/letsencrypt/live/$DOMAIN" ]]; then
      echo "Certificate lineage already exists for $DOMAIN — skipping issuance (per D-6)"
    elif [[ "${SKIP_CERTBOT:-0}" == "1" ]]; then
      echo "SKIP_CERTBOT=1 — skipping certbot issuance"
    else
      echo "Requesting initial TLS certificate for $DOMAIN..."
      certbot certonly \
        --authenticator webroot \
        --installer none \
        --no-eff-email \
        --agree-tos \
        --email "$ADMIN_EMAIL" \
        --webroot-path "$WEBROOT_PATH" \
        -d "$DOMAIN"
    fi

    SECTION 5 — certbot deploy hook (per D-4):
    mkdir -p /etc/letsencrypt/renewal-hooks/deploy
    cp "$REPO_ROOT/config/systemd/certbot-deploy-hook.sh" \
       /etc/letsencrypt/renewal-hooks/deploy/reload-nginx.sh
    chmod +x /etc/letsencrypt/renewal-hooks/deploy/reload-nginx.sh

    SECTION 6 — systemd OnFailure drop-in (per D-5):
    mkdir -p /etc/systemd/system/certbot.service.d
    cp "$REPO_ROOT/config/systemd/certbot.service.d/onfailure.conf" \
       /etc/systemd/system/certbot.service.d/onfailure.conf
    cp "$REPO_ROOT/config/systemd/certbot-renew-failure.service" \
       /etc/systemd/system/certbot-renew-failure.service
    systemctl daemon-reload || { echo "FATAL: systemctl daemon-reload failed — OnFailure drop-in not wired" >&2; exit 1; }
    systemctl is-enabled certbot.timer && echo "Stock certbot.timer is active — renewal schedule preserved"

    SECTION 7 — ufw firewall rules (per D-7, EDGE-04):
    Firewall section is guarded by SKIP_UFW:
    if [[ "${SKIP_UFW}" == "1" ]]; then
      echo "SKIP_UFW=1 — skipping ufw changes"
    else
      ufw default deny incoming || true
      ufw default allow outgoing || true
      ufw allow 22/tcp comment 'SSH operator access' || true
      ufw allow 80/tcp comment 'HTTP public + ACME challenges' || true
      ufw allow 443/tcp comment 'HTTPS public' || true

      # wg0 pre-check: refuse to apply the 6443 rule if WireGuard tunnel is not up.
      # Without the interface qualifier the k3s API would be exposed on the public interface.
      if ip link show wg0 >/dev/null 2>&1; then
        ufw allow in on wg0 to any port 6443/tcp comment 'k3s API via WireGuard only' || { echo "FATAL: ufw 6443/wg0 rule failed" >&2; exit 1; }
      else
        echo "FATAL: wg0 not found — refusing to apply firewall (6443 would be publicly exposed); bring up the WireGuard tunnel first" >&2
        exit 1
      fi

      ufw --force enable || { echo "FATAL: ufw --force enable failed" >&2; exit 1; }
      echo "Firewall rules:"
      ufw status verbose
    fi

    SECTION 8 — Operator verification reminder:
    echo "=== Bootstrap complete ==="
    echo "OPERATOR VERIFICATION REQUIRED (live host only — not CI):"
    echo "  1. nginx -t                             (validate nginx config)"
    echo "  2. certbot renew --dry-run              (verify renewal pipeline)"
    echo "  3. systemctl show -p OnFailure certbot.service  (confirm drop-in wired)"
    echo "  4. ufw status verbose                   (confirm split-tunnel rules)"
    echo "  5. curl -I https://$DOMAIN/             (smoke check public HTTPS)"
    echo "  See: docs/edge-bootstrap.md for full runbook"

    Critical idempotency constraints:
    - mkdir -p everywhere (never bare mkdir).
    - ln -sf (force-overwrite).
    - ufw allow 22/80/443 calls have || true (idempotent: rule already present is harmless).
    - ufw allow in on wg0 to any port 6443/tcp has NO || true — failure exits 1 (safety gate).
    - The 6443 rule is guarded by an "ip link show wg0" pre-check — script aborts if wg0 is absent.
    - ufw --force enable is error-checked with || { ... exit 1; } — not silently masked.
    - Section 4 checks lineage existence before cert issuance.
    - Backup written only once (check for .bak before cp).
    - cp overwrites config files idempotently.
    - systemctl daemon-reload is error-checked — not silently masked.
    - nginx -t is re-run after .bak restore to validate before any reload.

    After creation, chmod +x scripts/bootstrap-edge.sh.
  </action>

  <action>
    Create scripts/teardown-edge.sh. This script reverses every bootstrap step in reverse order,
    restoring the host to its pre-Phase-7 state (per D-8). The backed-up original vhost is
    restored from the .bak file so the live edge is exactly as it was before bootstrap.

    Header:
    #!/usr/bin/env bash
    set -euo pipefail

    Header comment block:
    # teardown-edge.sh — reverses bootstrap-edge.sh; restores host to pre-Phase-7 state
    # Restores the backed-up original vhost, removes drop-in/hook/ufw edge rules.
    # IMPORTANT: Does NOT remove /etc/letsencrypt/ certs (preserved for re-bootstrap).
    #            Does NOT remove ufw allow 22/tcp (would lock out operator).
    #            Does NOT disable ufw itself (firewall remains enabled).
    # Usage: scripts/teardown-edge.sh
    #        DOMAIN=stats-staging.solid-stats.ru scripts/teardown-edge.sh

    Variable defaults:
    : "${DOMAIN:=stats-staging.solid-stats.ru}"
    : "${NGINX_SITES_DIR:=/etc/nginx/sites-available}"
    : "${NGINX_SITES_ENABLED:=/etc/nginx/sites-enabled}"

    VHOST_CONF="$NGINX_SITES_DIR/stats-staging-solid-stats.conf"
    BAK_VHOST="${VHOST_CONF}.bak"

    Section structure:

    SECTION 1 — Remove systemd OnFailure drop-in (per D-5):
    systemctl stop certbot-renew-failure.service 2>/dev/null || true
    rm -f /etc/systemd/system/certbot.service.d/onfailure.conf
    rm -f /etc/systemd/system/certbot-renew-failure.service
    # Remove the drop-in dir if empty
    rmdir /etc/systemd/system/certbot.service.d 2>/dev/null || true
    systemctl daemon-reload
    echo "certbot.service.d drop-in and failure handler removed"

    SECTION 2 — Remove certbot deploy hook:
    rm -f /etc/letsencrypt/renewal-hooks/deploy/reload-nginx.sh
    echo "certbot deploy hook removed"

    SECTION 3 — Restore original nginx vhost (per D-8):
    if [[ -f "$BAK_VHOST" ]]; then
      echo "Restoring original vhost from $BAK_VHOST..."
      cp "$BAK_VHOST" "$VHOST_CONF"
      rm -f "$BAK_VHOST"
      echo "Original vhost restored; backup removed"
    else
      echo "No backup found at $BAK_VHOST — removing repo vhost"
      rm -f "$NGINX_SITES_ENABLED/stats-staging-solid-stats.conf"
      rm -f "$VHOST_CONF"
    fi
    # Re-enable symlink if vhost still present (backup was restored)
    if [[ -f "$VHOST_CONF" ]]; then
      ln -sf "$VHOST_CONF" "$NGINX_SITES_ENABLED/stats-staging-solid-stats.conf"
    fi
    if nginx -t 2>/dev/null; then
      systemctl reload nginx
      echo "nginx reloaded with restored config"
    else
      echo "warn: nginx -t failed after vhost restore — reload skipped; fix manually"
    fi

    SECTION 4 — Remove ufw edge rules:
    ufw delete allow 80/tcp 2>/dev/null || echo "ufw rule 80/tcp not found — skipping"
    ufw delete allow 443/tcp 2>/dev/null || echo "ufw rule 443/tcp not found — skipping"
    ufw delete allow in on wg0 to any port 6443/tcp 2>/dev/null || echo "ufw rule 6443/wg0 not found — skipping"
    # DO NOT remove ufw allow 22/tcp (operator lockout risk)
    # DO NOT run ufw disable (firewall remains enabled with remaining rules)
    echo "ufw edge rules removed (SSH rule preserved, firewall remains enabled)"
    ufw status verbose

    SECTION 5 — Summary:
    echo "=== Teardown complete ==="
    echo "Removed: certbot.service.d drop-in, failure handler, deploy hook, ufw 80/443/6443-wg0"
    echo "Restored: original nginx vhost from .bak (or removed if no backup)"
    echo "Preserved: /etc/letsencrypt/ certs, ufw 22/tcp SSH rule, ufw enabled"
    echo "OPERATOR VERIFICATION:"
    echo "  1. nginx -t                    (config valid without repo vhost)"
    echo "  2. ufw status verbose          (confirm 80/443/6443 rules removed)"
    echo "  3. systemctl list-timers       (stock certbot.timer still present)"
    echo "  4. systemctl show -p OnFailure certbot.service  (drop-in removed; OnFailure empty)"

    Critical safety constraints:
    - Never remove ufw allow 22/tcp.
    - Never run ufw disable.
    - All rm -f calls handle missing-file case with || echo "..." or 2>/dev/null.
    - systemctl stop has || true to handle not-running state.
    - Backup restoration uses cp then rm -f the .bak (atomic-ish on the same filesystem).

    After creation, chmod +x scripts/teardown-edge.sh.
  </action>

| Boundary | Description |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-7-03-01 | Elevation of Privilege | ufw 6443 exposed publicly (missing interface qualifier or wg0 absent) | mitigate | Rule is "ufw allow in on wg0 to any port 6443/tcp"; bootstrap pre-checks "ip link show wg0" before applying the rule — exits 1 with FATAL if wg0 is absent so the rule is never silently skipped; the 6443 rule has no "|| true" masking and errors hard; validate-edge.py checks for the full literal "ufw allow in on wg0 to any port 6443" (not just "wg0"); without the wg0 qualifier the k3s API would be public |
| T-7-03-02 | Denial of Service | bootstrap locks out operator via ufw misconfiguration | mitigate | ufw allow 22/tcp applied before ufw --force enable; teardown never removes 22/tcp; firewall never disabled during teardown; SSH access preserved through both ops |
| T-7-03-03 | Availability | live vhost disrupted during adopt (nginx -t failure) | mitigate | bootstrap backs up .bak before overwrite; if nginx -t fails after install, restores backup and exits 1; nginx never reloaded on invalid config |
| T-7-03-04 | Availability | cert re-issuance during bootstrap when lineage exists | mitigate | Section 4 checks /etc/letsencrypt/live/$DOMAIN before certbot certonly; existing lineage skips issuance (per D-6, D-8); SKIP_CERTBOT=1 override available |
| T-7-03-05 | Tampering | relay/auth vhosts accidentally affected by bootstrap | mitigate | Bootstrap only operates on stats-staging-solid-stats.conf; no glob or wildcard on sites-available; other vhosts (sg-stats-relay.conf, default) untouched (per D-1) |
| T-7-03-06 | Availability | firewall misorder — deny applied before allow (lockout) | mitigate | ufw sets defaults (deny incoming, allow outgoing) first; specific allow rules follow; ufw --force enable is last; script exits on any error (set -euo pipefail); SSH allow is first specific rule |
| T-7-03-SC | Tampering | apt installs (certbot, nginx, ufw, curl, openssl) | accept | All packages are Ubuntu 24.04 apt standard packages; approved in Package Legitimacy Audit in 07-RESEARCH.md; no pip/npm/cargo |
</threat_model>
