    File header comment block:
    # stats-staging-solid-stats.conf — nginx vhost for stats-staging.solid-stats.ru
    # Managed by: infrastructure repo (config/nginx/sites-available/)
    # Applied by: scripts/bootstrap-edge.sh (operator-run, not CI)
    # Live file: /etc/nginx/sites-available/stats-staging-solid-stats.conf (symlinked into sites-enabled)
    # Phase 11 cutover lever: the upstream server line marked # CUTOVER: below
    # Do NOT edit this file directly on the host — changes are overwritten on re-bootstrap.

    Named upstream block (MUST appear before the server blocks, per D-2, D-3):
    upstream solid_stats_staging_server2 {
        # CUTOVER: change this server address to switch production traffic (Phase 11 lever)
        # Current: k3s server-2 ClusterIP (routable on the VPS host node)
        server 10.43.94.103:3000;
        keepalive 16;
    }

    Block 1 — HTTP (port 80):
    # HTTP: ACME challenge path + redirect to HTTPS
    server {
        listen 80;
        listen [::]:80;
        server_name stats-staging.solid-stats.ru;

        # ACME webroot for certbot — do NOT proxy this to the upstream
        location /.well-known/acme-challenge/ {
            root /var/www/html;
        }

        location / {
            return 301 https://$host$request_uri;
        }
    }

    Block 2 — HTTPS (port 443):
    # HTTPS: TLS termination + proxy to k3s server-2 upstream
    server {
        listen 443 ssl http2;
        listen [::]:443 ssl http2;
        server_name stats-staging.solid-stats.ru;

        ssl_certificate /etc/letsencrypt/live/stats-staging.solid-stats.ru/fullchain.pem;
        ssl_certificate_key /etc/letsencrypt/live/stats-staging.solid-stats.ru/privkey.pem;
        include /etc/letsencrypt/options-ssl-nginx.conf;
        ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

        add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

        location / {
            proxy_pass http://solid_stats_staging_server2;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_http_version 1.1;
            proxy_set_header Connection "";
            proxy_connect_timeout 60s;
            proxy_send_timeout 60s;
            proxy_read_timeout 60s;
        }
    }

    Key constraints:
    - Use include /etc/letsencrypt/options-ssl-nginx.conf and ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem (certbot-managed, per D-2), NOT hand-rolled ssl_protocols/ssl_ciphers lines — the certbot include owns those.
    - ssl_certificate and ssl_certificate_key path is /etc/letsencrypt/live/stats-staging.solid-stats.ru/{fullchain,privkey}.pem (per D-2).
    - The 443 listen directives MUST be "listen 443 ssl http2;" and "listen [::]:443 ssl http2;" — the live vhost uses http2 (confirmed via SSH capture: "listen 443 ssl http2;"). Omitting http2 would DOWNGRADE the live edge when the repo copy is applied (FIX 5).
    - The "# CUTOVER:" comment must be on the line immediately before or on the server line inside the upstream block so Phase 11 can locate it with grep.
    - The upstream name solid_stats_staging_server2 must be used consistently in both the upstream block and the proxy_pass directive.
