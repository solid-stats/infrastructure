    # Edge Bootstrap: nginx Vhost, TLS Renewal Hook, and Firewall

    Intro paragraph (2-3 sentences): This is the operator runbook for adopting the existing
    staging host edge (nginx vhost, certbot renewal hook, host firewall) into the infrastructure
    repo. The bootstrap script is idempotent and backs up the live vhost before applying the repo
    copy. Live verification steps require SSH access to the VPS; offline structural checks run
    in CI via scripts/validate-edge.py.

    ## Context: Adopt, Not Rebuild

    Short paragraph: The staging edge (stats-staging.solid-stats.ru) is already live and serving
    traffic. Phase 7 does NOT rebuild it — it:
    1. Stores the live vhost config verbatim in the repo (config/nginx/sites-available/stats-staging-solid-stats.conf).
    2. Adds a certbot deploy hook (nginx -t gate before reload, installed to /etc/letsencrypt/renewal-hooks/deploy/).
    3. Adds an OnFailure= drop-in on the stock certbot.service to surface renewal failures to journald.
    4. Applies ufw split-tunnel firewall rules (80/443 public, 6443 via WireGuard only).
    5. Keeps the existing certbot.timer — no new renewal timer is created.

    ## Prerequisites

    - SSH access to the staging VPS as root or with sudo.
    - Git clone of this repository on the VPS (or copy scripts/ and config/ to the host).
    - ADMIN_EMAIL — a valid email for Let's Encrypt (used only if no cert lineage exists; the
      cert is already issued, so bootstrap skips issuance by default).
    - WireGuard wg0 interface up on the VPS (from Phase 6 operator-bootstrap.md).
    - Port 80 and 443 inbound reachable from the public internet.

    ## Offline Checks (CI — no VPS required)

    Note: OFFLINE-VERIFIABLE. These checks run in CI without touching the VPS. They validate
    repo artifact structure but NOT live nginx or certbot behavior.

    ```
    python3 scripts/validate-edge.py
    ```

    Expected output: five "ok:" lines (nginx vhost shape, shell scripts, systemd drop-in shape,
    bootstrap idempotency markers, teardown script). One "warn:" about nginx -t being operator-only.

    ## Step 1: Clone / update the repo on the VPS

    ```bash
    git pull   # or git clone https://github.com/... if first time
    ```

    The repo must be current so bootstrap-edge.sh installs the latest config files.

    ## Step 2: Run the bootstrap script

    ```bash
    ADMIN_EMAIL=your@email.com scripts/bootstrap-edge.sh
    ```

    What bootstrap does:
    - Installs packages (certbot, nginx, ufw, curl, openssl).
    - Backs up the live /etc/nginx/sites-available/stats-staging-solid-stats.conf to a .bak file.
    - Installs the repo vhost, runs nginx -t gate, reloads nginx.
    - Skips cert issuance (lineage already exists for stats-staging.solid-stats.ru).
    - Installs the certbot deploy hook to /etc/letsencrypt/renewal-hooks/deploy/reload-nginx.sh.
    - Installs the OnFailure= drop-in to /etc/systemd/system/certbot.service.d/onfailure.conf.
    - Applies ufw rules (22/tcp, 80/tcp, 443/tcp, 6443/tcp on wg0 only).

    To skip ufw changes (e.g. ufw already configured):
    ```bash
    ADMIN_EMAIL=your@email.com SKIP_UFW=1 scripts/bootstrap-edge.sh
    ```

    The script is safe to re-run. Second run produces the same final state as the first.

    ## Step 3: Operator-Only Live Verification

    All checks in this section are OPERATOR-ONLY — require live VPS access.

    ### 3a. nginx config syntax — OPERATOR-ONLY
    ```bash
    nginx -t
    ```
    Expected: "syntax is ok" and "test is successful".

    ### 3b. Certbot renewal dry-run — OPERATOR-ONLY
    ```bash
    certbot renew --dry-run
    ```
    Expected: "Congratulations, all simulated renewals succeeded" or "No renewals were attempted"
    (if cert is not due for renewal). This confirms the webroot path, the deploy hook, and ACME
    connectivity all work.

    ### 3c. OnFailure= drop-in wired — OPERATOR-ONLY
    ```bash
    systemctl show -p OnFailure certbot.service
    ```
    Expected: OnFailure=certbot-renew-failure.service. If empty, daemon-reload was not run —
    repeat: systemctl daemon-reload.

    ### 3d. Firewall split-tunnel rules — OPERATOR-ONLY
    ```bash
    ufw status verbose
    ```
    Expected: 22/tcp ALLOW Anywhere, 80/tcp ALLOW Anywhere, 443/tcp ALLOW Anywhere,
    6443/tcp on wg0 ALLOW Anywhere. The 6443 rule MUST include "on wg0" — without it the k3s
    API would be publicly reachable.

    ### 3e. Public HTTPS smoke check — OPERATOR-ONLY
    ```bash
    curl -I https://stats-staging.solid-stats.ru/
    ```
    Expected: HTTP/1.1 200 or HTTP/2 200 with a valid TLS handshake (no certificate errors).

    ## Step 4: Certificate Renewal Verification

    Note: OPERATOR-ONLY. The stock certbot.timer already runs twice daily. To verify the
    renewal pipeline:

    ```bash
    certbot renew --dry-run
    ```

    To check for past renewal failures:
    ```bash
    journalctl -t certbot-alert --since today
    ```
    No output means no failures today. Any entry indicates a renewal failure — inspect with:
    ```bash
    journalctl -u certbot.service -n 20
    ```

    Note: Do NOT create a custom certbot-renew.timer — the stock certbot.timer is already
    running and is the correct renewal mechanism. Phase 7 extends it only with a drop-in.

    ## Phase 11 Cutover Lever

    The nginx upstream is the Phase 11 traffic cutover lever. The exact location is in:

    ```
    config/nginx/sites-available/stats-staging-solid-stats.conf
    ```

    The upstream block:
    ```nginx
    upstream solid_stats_staging_server2 {
        # CUTOVER: change this server address to switch production traffic (Phase 11 lever)
        server 10.43.94.103:3000;
        keepalive 16;
    }
    ```

    To switch traffic to a different runtime: edit the `server` line inside the upstream block
    (the line marked `# CUTOVER:`), then re-run bootstrap-edge.sh (or run nginx -t && systemctl
    reload nginx manually). Phase 11 documents this step in detail.

    ## Reversibility: Teardown

    Before Phase 11, the operator must prove the edge is reversible in isolation. Run teardown
    on the VPS to undo all bootstrap steps:

    ```bash
    scripts/teardown-edge.sh
    ```

    This removes:
    - The certbot.service.d/onfailure.conf drop-in and certbot-renew-failure.service.
    - The certbot deploy hook at /etc/letsencrypt/renewal-hooks/deploy/reload-nginx.sh.
    - The repo vhost from sites-available and sites-enabled; restores the original from the .bak backup.
    - ufw rules for 80/tcp, 443/tcp, and 6443/tcp on wg0.

    Preserved (NOT removed):
    - /etc/letsencrypt/ certificates (run certbot delete -d stats-staging.solid-stats.ru explicitly).
    - ufw allow 22/tcp SSH rule.
    - ufw firewall itself (remains enabled).

    After teardown, verify (OPERATOR-ONLY):
    ```bash
    ufw status verbose             # 80/443/6443 rules absent
    systemctl list-timers          # stock certbot.timer still active (not removed by teardown)
    systemctl show -p OnFailure certbot.service  # OnFailure field now empty
    nginx -t                       # nginx config valid with restored original vhost
    ```

    ## Troubleshooting

    - "FATAL: ADMIN_EMAIL is required": set ADMIN_EMAIL before running bootstrap-edge.sh.
    - "FATAL: nginx config invalid after vhost install": the repo vhost has a syntax error or
      the certbot TLS files are missing; the script restores the .bak backup automatically;
      fix the vhost and re-run.
    - "certbot renew --dry-run: failed to connect": port 80 not reachable from internet —
      verify Timeweb perimeter firewall allows port 80 inbound.
    - "ufw rule 6443 without 'on wg0'": this exposes the k3s API publicly — remove with
      ufw delete allow 6443/tcp and re-run bootstrap-edge.sh.
    - "OnFailure= field empty after daemon-reload": drop-in dir name must be certbot.service.d
      (not certbot-renew.service.d); check /etc/systemd/system/certbot.service.d/onfailure.conf exists.
    - "WireGuard wg0 not found (ufw rule silent)": complete Phase 6 operator-bootstrap first.

    See also: docs/operator-bootstrap.md (Phase 6 WireGuard + RBAC bootstrap).
